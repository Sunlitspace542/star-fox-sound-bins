# Star Fox Sound Binary Disassemblies
This repository contains disassemblies of all sound binaries (SBNs) used in Star Fox, except for the sound driver (SGSOUND0).  
For a disassembly of SGSOUND0, see [Phonymike's disassembly](https://github.com/phonymike/starfox_spc_driver).  

Thanks to MrL314 for his BIN splitter script and Phonymike for figuring out what was in each BIN ages ago.
## Assembly
Either run ``build.bat``, or run ``make`` on the command line.  
You can edit the makefile to specify which BINs to build, rather than all of them by changing ``$(BUILD_FILES)`` in the ``BUILD ?=`` option to the names of the BINs you wish to assemble.  
Assembled output will appear in the ``build`` directory.  

## Notes on modifying
- For making custom instruments, you can use [this calculator](https://livvy94.neocities.org/cool/ADSR) to compute ADSR values for your new patch table entries.

- If you're using .BRR files from SMWCentral or generated by something like C700VST, use ``INC_SMPL2`` and ``SMPL_PTR2`` instead of ``INC_SMPL`` and ``SMPL_PTR``. These "internet" .BRR files include the loop point as the first two bytes. These alternative macros account for that.

- For SMWCentral sample packs, you can use the ``#instruments`` section in the AddMusicK .txt file in the patch table. Replace the sample filename with the index of the sample in the sample directory, and format the bytes as a ``db`` statement like what's already in the assembly. e.g. ``"Overdriven Guitar.brr" $FF $E0 $B8 $07 $90`` becomes ``db $xx, $FF, $E0, $B8, $07, $90 ; $XX

- [EBMusEd4SF](https://github.com/Sunlitspace542/ebmused4sf) is a fork of EBMusEd designed to be able to work with the files in this repository (e.g. assembled instrument SBNs like SGSOUND2/3, song data BINs).
